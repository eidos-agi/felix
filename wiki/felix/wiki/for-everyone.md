# Felix For Everyone

Felix is public infrastructure for agent ecosystems.

The Eidos stack is the reference implementation, not the product boundary.

## Portable Language

Felix should say:

- wiki/docs before Scridos
- router/orchestrator before Reeves
- open-source health before FOSS Forge
- repo visibility before private/public default
- identity image prompt before generated asset

## Adapter Rule

Eidos tools should be adapters:

- Scridos adapter for repo-native wiki/tasks
- FOSS Forge adapter for open-source health
- Reeves adapter for routing/memory

## Counter-Arguments and Gaps

If Felix becomes too generic, it will stop being useful. The balance is: generic contract, concrete default adapter.
