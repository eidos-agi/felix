# Add Setup-First Principle

Agent setup is the highest-leverage decision in the stack. Great downstream behavior flows from the initial choices about ontology, autonomy boundary, coupling, memory substrate, source-of-truth topology, run mode, and proof loop.

## Done Evidence

`felix interview` now names setup as the highest-leverage decision before role-boundary questions.

`felix scaffold-plan` now tells maintainers to decide ontology, autonomy, coupling, memory, source-of-truth, run mode, and proof before writing files.

The public README, standards, and examples now document setup-first discipline.
